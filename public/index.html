Get Started

The Sync-Workbench API provides programmatic access to read and manage employee data. You can retrieve employee details, manage roles, track performance, and more.

To use this API, you need an API key. Please contact us at support@sync-workbench.com to get your own API key.

get characters

To get characters you need to make a POST call to the following url :
http://api.westeros.com/character/get


QUERY PARAMETERS

Field Type Description
api_key String The unique API key provided to authenticate and authorize your API requests. Ensure that it is kept secure and not exposed publicly.
status String (optional) Filters employees based on their status. Acceptable values:
active - for employees who are currently active,
inactive - for employees who are inactive or no longer part of the system. If not provided, no filtering by status is applied.
page Integer (optional - default: 1) The page number for pagination. Used to retrieve a specific set of results based on the current page. If not provided, the first page is returned by default.
limit Integer (optional - default: 10) The maximum number of records to return in a single response. The value must be between 1 and 100. It helps control the size of the response for large datasets.
sort_type String (optional) The sorting order of the results. Acceptable values:
asc for ascending order (smallest to largest),
desc for descending order (largest to smallest). If not provided, the results will not be sorted.
sort_by String (optional) The field by which the results will be sorted. For example, 'updatedAt' to sort by the most recent updates, or 'firstName' to sort alphabetically by employee name. If not provided, no sorting is applied.
populate String (optional) Specifies related properties to populate along with the employee data. For example, 'location' or 'role'. If not provided, only primary employee data is returned.
role String (optional) Filters employees by their role within the organization. Acceptable values depend on your system's roles and the structure of your employee data. If not provided, all employee roles will be included.

Errors

Each error response follows the structure defined below:


{
  "status": "error",
  "statusCode": 400,
  "error": {
    "code": "BAD_REQUEST",
    "message": "Validation failed for input data.",
    "details": [
      {
        "code": "invalid_type",
        "expected": "string",
        "received": "number",
        "path": ["username"],
        "message": "Expected string, received number"
      },
      {
        "code": "too_small",
        "minimum": 1,
        "type": "string",
        "inclusive": true,
        "path": ["password"],
        "message": "String must contain at least 1 character"
      }
    ],
    "suggestion": "Please check the input fields and ensure they meet the required criteria."
  },
  "requestId": "123e4567-e89b-12d3-a456-426614174000",
  "documentation_url": "https://api.example.com/docs/errors#BAD_REQUEST",
  "timestamp": "2024-12-24T12:00:00Z"
}

The Sync-Workbench API uses the following error codes:

Status Code Type Meaning
SYNTAX_ERROR Invalid JSON format. Invalid JSON format in request body. Check for syntax errors like missing quotes or extra commas. Ensure JSON keys are double-quoted and there are no trailing commas.
BAD_REQUEST Validation failed for input data. Validation failed for the input data. Please check the input fields and try again.
INTERNAL_SERVER_ERROR An unexpected server error occurred. Unexpected error occurred on the server. Please try again later or contact support if the issue persists.
NOT_FOUND Resource not found. The requested resource could not be found on this server. Please check the URL or refer to the API documentation.
AUTHENTICATION_FAILED Invalid credentials. Authentication failed due to invalid credentials. Please verify your credentials and try again.
PERMISSION_DENIED Permission denied. You do not have permission to access this resource. Please check your permissions or contact the administrator.
RESOURCE_CONFLICT Resource already exists. The specified resource already exists. Please use a unique identifier or check the resource for correctness.
UNPROCESSABLE_ENTITY Semantic errors in request. The request could not be processed due to semantic errors. Please verify the input data and try again.
METHOD_NOT_ALLOWED Method not allowed for endpoint. The HTTP method used is not allowed for this endpoint. Please check the allowed methods for this endpoint.
TOO_MANY_REQUESTS Rate limiting exceeded. Too many requests sent in a short period. Please wait before sending more requests.
